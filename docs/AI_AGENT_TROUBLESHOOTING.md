# Почему ИИ-агент «молчит» после второго сообщения

## Что вы видели

1. **Вы:** «привет, нужна запись»  
2. **Бот:** «Привет»  
3. **Вы:** «нужна запись на коррекцию»  
4. **Бот:** тишина.

То есть первое сообщение получило короткий ответ, второе — никакого.

---

## В чём причина

Цепочка такая:

1. **Orchestrator** получает сообщение, через ~20 сек (debounce) вызывает **AI**, получает ответ (например, «RESPOND» + текст).
2. Чтобы отправить ответ клиенту, оркестратор дергает **Gateway**: `POST /whatsapp/send`.
3. **Gateway** отправляет сообщение только через **WhatsApp Web** (whatsapp-web.js). Если сессия не готова — возвращает **503** «WhatsApp Web not ready».
4. Оркестратор при **любом не-2xx** (в т.ч. 503) считает отправку неудачной и **ничего не пишет в чат** и не падает с ошибкой — получается «молчит».

Итого: **чаще всего «молчит» из‑за того, что в момент отправки второго ответа WhatsApp Web в Gateway не был готов** (перезапуск контейнера, потеря сессии, блокировка Chromium и т.п.). Первый ответ мог уйти, пока сессия ещё была жива; второй — уже нет.

---

## Что проверить

### 1. Логи оркестратора

После включённого логирования при ошибке отправки вы увидите что-то вроде:

```text
WhatsApp send failed: gateway returned non-2xx (e.g. 503 = WhatsApp Web not ready)
status: 503, toPhone: +43...
```

Если такое есть — оркестратор ответ сгенерировал, но Gateway не смог отправить.

### 2. Логи Gateway

- Ошибки Chromium/«profile in use» → сессия WhatsApp Web не поднялась.
- Сообщения про «WhatsApp Web client ready» / «authenticated» → сессия в порядке.

### 3. Сессия WhatsApp Web

- Один раз нужно отсканировать QR (например, `scripts/get-wa-qr.sh`) и держать сессию.
- После перезапуска контейнера Gateway часто появляется блокировка профиля Chromium — тогда нужно снять lock-файлы и перезапустить Gateway (см. runbook).

---

## Что сделано в коде

- При неудачной отправке (не 2xx от Gateway) оркестратор пишет в лог предупреждение с `status` и кратким телом ответа, чтобы по логам было видно «отправка не прошла, скорее всего 503».

---

## Краткий чеклист

| Шаг | Действие |
|-----|----------|
| 1 | Убедиться, что WhatsApp Web в Gateway подключён (логи «client ready» / нет 503 при отправке). |
| 2 | При «profile in use» — удалить lock-файлы в volume и перезапустить Gateway. |
| 3 | Пересобрать/перезапустить оркестратор и отправить тестовое второе сообщение. |
| 4 | В логах оркестратора искать «WhatsApp send failed» — если есть, причина в Gateway/WhatsApp Web. |
